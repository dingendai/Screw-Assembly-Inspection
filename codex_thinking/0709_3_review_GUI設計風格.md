# GUI 設計風格與用色說明

日期：2026-07-09

## 目的

本文件整理目前桌面 GUI 的設計風格、色彩邏輯、元件語言與導覽結構，作為後續 UI 調整、風格延伸與文件說明的基礎。

分析依據主要來自：

- `app/src/valve_gui/styles.py`
- `app/src/valve_gui/main_window.py`

## 設計定位

目前這套 GUI 的風格不是消費型 App，也不是行銷導向頁面，而是偏向：

```text
淺色工業控制台 + 企業內部作業系統 + 現場設備管理介面
```

整體語言強調：

- 長時間操作時的可讀性
- 功能分區清楚
- 狀態辨識效率高
- 表單、表格、相機畫面、設定頁可穩定共存

它比較接近 HMI、設備監控系統或品管後台，而不是一般品牌官網式 UI。

## 整體視覺風格

### 1. 背景基調

主背景不是純白，而是暖灰米白色，減少長時間觀看的刺眼感，也讓系統更像工業管理介面。

主背景色：

```text
#f1eee7
```

這個底色讓整體畫面看起來乾淨、安定、偏務實，而不是冷冽或炫技。

### 2. 結構感

介面以卡片、群組框、工具列、表格為主，區塊分明，圓角控制保守，明顯偏向工程與管理系統的語氣。

特徵包括：

- 白底卡片
- 淺灰藍邊框
- 低裝飾性
- 以功能層級而不是插畫或大圖建立視覺節奏

### 3. 操作語氣

互動設計不是靠大量動畫，而是靠：

- 顏色差異
- 邊框狀態
- 粗體文字
- 區塊層次

這種做法很適合現場操作與品管判讀，不會因為花俏效果干擾使用。

## 色彩系統

這套色彩不是單一主色壓到底，而是明確做了角色分工。

### 1. 基礎色

主背景：

```text
#f1eee7
```

主文字：

```text
#172026
```

次文字：

```text
#41515c
#687981
```

說明：

- `#172026` 取代純黑，閱讀壓力較小
- `#41515c` 用於標題與中階說明
- `#687981` 用於弱提示與次要資訊

### 2. 主操作色

主操作按鈕、關鍵確認、主要選中狀態，使用墨綠色系：

```text
#176b5d
#135b50
```

說明：

- `#176b5d` 是主功能色
- `#135b50` 是 hover 或加深狀態
- 這組顏色帶有穩定、專業、設備控制感，不會過度活潑

### 3. 設定區色彩

`S1~S6` 類設定區導覽按鈕使用中性淺灰：

```text
#eef2f4
#e2e8ec
#d6dde1
#c8d1d6
```

說明：

- 色彩語氣偏理性、工程化
- 不搶主操作色
- 適合作為設定類、結構類、系統參數類操作入口

### 4. 資訊 / 品管區色彩

資訊類與品管類導覽使用淺藍色系：

```text
#e0f2fe
#bae6fd
#7dd3fc
#082f49
```

說明：

- 與設定區的灰色區分開
- 提供資訊型、統計型、查詢型頁面的視覺身份
- 藍色比綠色更像資料與分析，而不是執行操作

### 5. 危險 / 登出色

登出與風險操作使用低飽和紅色：

```text
#ffe2df
#ffd2cc
#f4aaa4
#e98f86
#9f1f16
```

說明：

- 不是警報級鮮紅
- 是「應留意」而非「立即緊急」的控制色
- 非常適合登出、刪除、取消等風險操作

### 6. 選中強調色

目前導覽類按鈕的選中狀態主要用紅色邊框強調：

```text
#dc2626
#b91c1c
```

說明：

- 用紅框而不是整片深色填滿，辨識很快
- 能明確告訴使用者目前所在頁面
- 在淺色系 UI 裡辨識度高

## 狀態色規則

系統對結果與狀態的顏色表達非常明確。

### PASS / 成功

```text
背景：#dff4e9
文字：#0d6b3f
邊框：#8fd7b3
```

語氣：

- 正向
- 安全
- 品質通過

### NG / 異常

```text
背景：#ffe2df
文字：#b42318
邊框：#f4aaa4
```

語氣：

- 異常
- 不合格
- 需立即注意

### 等待 / 中性

```text
背景：#e6ecef
文字：#56656d
```

語氣：

- 尚未完成
- 中性狀態
- 尚未產生結果

### 警示框 / 原因區

```text
背景：#fff8e7
文字：#6f4e00
邊框：#f0d68a
```

語氣：

- 說明型警示
- 非致命但要理解原因
- 適合顯示 NG 原因或提示訊息

## 字體系統

目前字體設定為：

```text
"Microsoft JhengHei UI", "Segoe UI", Arial
```

設計語氣：

- 完全偏 Windows 原生管理系統
- 不追求品牌展示感
- 優先確保中文可讀性與系統穩定性

字重使用特徵：

- 標題常用 `700` 或 `800`
- 狀態字與 badge 偏粗
- 一般內容與設定欄位維持正常可讀性

這代表此系統是「操作優先」而不是「形象優先」。

## 元件語言

### 1. 圓角

主要圓角落在：

```text
6px ~ 8px
```

代表風格偏克制、務實，不走過度柔化的消費型 UI。

### 2. 邊框

常見邊框色：

```text
#d7dee2
#c9d4d9
#bcc8ce
```

特性：

- 低對比
- 穩定
- 讓表單、表格、卡片之間有清楚分界

### 3. 按鈕

按鈕不是極簡無邊框，而是明確有：

- 背景色
- 邊框
- hover 狀態
- 輕微陰影

這讓按鈕在管理型系統裡更容易辨識與點擊。

### 4. 卡片與群組

`QGroupBox` 與相機卡片採用：

- 白底
- 淺色邊框
- 小圓角
- 保守陰影

這個設計讓相機畫面、規則表格、設定群組都能穩定共存。

## 導覽結構

目前導覽概念已經形成三層：

### 1. 頂部工具列

定位：

- 主流程入口
- 全域操作
- 角色資訊
- 套用設定 / 進入監測這類高頻操作

### 2. 右側側欄

定位：

- 次級導覽
- 設定模組
- 品管模組
- 可收合工具列

### 3. 中央內容區

定位：

- 真正工作區
- `QStackedWidget` 對應各頁面
- 所有主要任務都在中央完成

這種結構很像企業工作台，而不是傳統網站導覽。

## 設計風格一句話定義

可以將這套 GUI 定義為：

> 以淺色工業控制台為核心的企業級檢測 GUI，強調長時間操作的可讀性、功能分區清晰度與狀態辨識效率，透過暖灰底、墨綠主操作色、淺藍資訊色與低飽和警示紅，建立穩定、專業且適合現場設備管理的視覺系統。
